1. FIX DAILY REMINDER NOTIFICATION SPAM

---

Current issue:
The daily reminder notification is sent multiple times (sometimes 10–16 times).

Fix the system so that the notification can only be sent once per day.

Create a new file:

frontend/src/lib/daily-reminder.ts

Implement the following:

1. Daily lock

Use localStorage key:

finradar-reminder-YYYY-MM-DD

If the key already exists, do not send another notification.

2. Cooldown protection

Store last reminder timestamp:

localStorage.setItem("finradar-last-reminder", Date.now())

Do not allow another reminder within 10 minutes.

3. Notification tag

When sending notifications use:

tag: "financial-radar-daily-reminder"

Example:

new Notification("Financial Radar", {
body: "Jangan lupa catat transaksi hari ini! 💰",
tag: "financial-radar-daily-reminder"
})

This prevents duplicate notifications stacking.

4. Prevent multiple schedulers

Ensure the reminder scheduler only runs once:

if (window.finRadarReminderStarted) return;
window.finRadarReminderStarted = true;

5. React scheduler safety

If the reminder runs in React useEffect, ensure:

useEffect(() => {
startDailyReminder()
}, [])

6. Respect user settings

If the "Pengingat transaksi harian" toggle is OFF, do not run the scheduler.

4. IMPLEMENT OFFLINE QUEUE SYSTEM

---

Create file:

frontend/src/lib/offline-sync.ts

Goal:
When user creates or edits a transaction while offline, store the action locally.

Flow:

User action
↓
navigator.onLine ?
YES → call API
NO  → store in offline_queue

When internet returns:

sync queue to server.

Implementation:

window.addEventListener("online", syncOfflineQueue)

Loop through offline_queue and send requests to:

/api/transactions

After successful sync remove items from queue.
