Upgrade the Financial Radar PWA to a fully offline-proof architecture.

Current problem:
The app partially works offline but fails when dynamically imported modules are required and some API calls still attempt network requests.

Goal:
The entire app must work offline with no runtime module errors.

Users must still be able to:

* create expense, income, transfer transactions
* scan receipts
* add accounts
* create savings goals
* record debt payments
* update budgets
* gain XP and streaks
* create custom categories
* navigate dashboard

All actions must work offline and sync automatically when internet returns.

IMPORTANT:
Do not modify backend code.
All work must be done in frontend.

---

1. REMOVE ALL DYNAMIC IMPORTS

---

Search the codebase for:

import("...")

and replace them with static imports.

Example:

WRONG

const { saveOfflineAction } = await import("./offline-sync")

CORRECT

import { saveOfflineAction } from "./offline-sync"

Reason:
Dynamic imports break when offline because Vite generates separate chunks that must be fetched.

3. OFFLINE QUEUE SYSTEM

---

frontend/src/lib/offline-sync.ts
