GOAL
Unify the transaction creation UX so the app always uses the same bottom sheet flow.

CURRENT PROBLEM
Right now there are multiple transaction entry UIs:

1. Global floating "+" button opens a bottom sheet with action selection (Expense / Income / Transfer / Saving / Pay Debt / Scan).
2. On the Transactions page, the green "+ Tambah Transaksi" button opens a different transaction form layout.
3. On the Transactions page, the floating "+" button opens another bottom sheet variant.

This creates inconsistent UX.

TARGET UX (Single Flow)

We want ONE unified bottom sheet pattern:

Step 1
Whenever the user presses ANY add button:

* Global floating "+"
* "+ Tambah Transaksi" button on Transactions page

Open the SAME bottom sheet:

"Action Selector Bottom Sheet"

Title:
"Apa yang ingin kamu lakukan?"

Options:

* Pengeluaran
* Pemasukan
* Transfer
* Tabungan
* Bayar Hutang
* Scan

This bottom sheet matches the design currently used in the floating "+" action (the clean selector layout).

Step 2
After the user selects one action (for example Pengeluaran), open the Transaction Form Bottom Sheet.

This form bottom sheet is the one shown in the correct UI version (the form that contains):

Fields:

* Tanggal
* Jumlah
* Dari Rekening
* Kategori
* Catatan

Buttons:

* Catat Transaksi
* Batal

Step 3
The Transactions page "+ Tambah Transaksi" button must trigger the SAME selector bottom sheet first, not directly open the form.

So the flow becomes:

User taps "+" anywhere
→ Action selector bottom sheet appears
→ User chooses action
→ Transaction form bottom sheet appears

IMPLEMENTATION RULES (VERY IMPORTANT)

DO NOT modify:

* API calls
* database logic
* validation logic
* transaction business logic
* mutation hooks
* react-query logic
* backend endpoints

Only modify UI routing and component usage.

REFACTOR INSTRUCTIONS

1. Identify all components that open transaction forms directly.
2. Replace them so they all call the same bottom sheet component (ActionSelectorBottomSheet).
3. When an option is selected, open the existing TransactionFormBottomSheet with the selected type.

Example structure:

ActionSelectorBottomSheet
→ user selects type
→ open TransactionFormBottomSheet(type)

4. The Transactions page "+ Tambah Transaksi" button must call the same openActionSelector() handler used by the global "+" FAB.

5. Remove duplicate transaction modal implementations if they exist.

EXPECTED RESULT

All entry points now behave consistently:

Global "+"
→ Action selector
→ Transaction form

Transactions page "+ Tambah Transaksi"
→ Action selector
→ Transaction form

The UI should match the clean bottom sheet style currently used in the selector design.

Do not change styling theme or spacing unless required to unify the component.
