Refactor the authentication flow of this project to support a **Guest-first experience with optional Google account linking**.

Current problem:
The app currently tries to manage guest login and Google login in a conflicting way which causes Google login to revert back to Guest.

Desired behavior:

1. DEFAULT USER STATE

* When a user opens the dashboard, they should automatically be treated as a **Guest user**.
* This guest state should exist only on the frontend unless they link their account.
* The UI should display:

  * Name: Guest
  * Level system still works
  * Data stored locally if needed

2. REMOVE LOGOUT BUTTON FOR GUEST

* Remove the logout button completely from both:

  * Desktop layout
  * Mobile layout
* Logout should only exist if a user is authenticated with Google.

3. GOOGLE LOGIN FLOW
   The only login mechanism should be via:

"Tautkan dengan Google"

When clicked:

* redirect to `/api/login`
* complete Google OAuth
* backend creates session
* frontend fetches `/api/auth/user`
* user replaces guest state

4. REMOVE AUTO GUEST LOGIN API
   If there is any endpoint like:

/api/guest-login
/api/auth/guest

Remove it.

Guest mode should be purely frontend state.

5. UPDATE use-auth.ts
   Modify the hook so that:

* It tries to fetch `/api/auth/user`
* If the request returns 401 → user remains Guest
* Do NOT automatically call guest login mutation.

Expected logic:

```
if authenticated → return user
if 401 → return null (frontend treats as Guest)
```

6. UI BEHAVIOR
   If user === null:

Show:

Guest
"Progress belum disimpan ke cloud"
"Tautkan dengan Google"

If user exists:

Show real profile info.

7. PROFILE PAGE
   Update the profile page logic:

If Guest:

* show "Tautkan dengan Google"
* hide logout button

If authenticated:

* show "Logout"

8. CLEAN AUTH STATE
   Ensure the following:

Guest state = frontend only
Authenticated state = session cookie

Never override authenticated user with guest.

9. MOBILE AND DESKTOP UI
   Apply the same behavior across:

* sidebar profile
* mobile bottom navigation
* profile page

10. FINAL RESULT

Guest experience:
User opens app → automatically Guest.

If user taps:

"Tautkan dengan Google"

then the app upgrades the guest session to a real account.

Do not break existing backend OAuth logic.
Only clean up the frontend auth logic and UI.
