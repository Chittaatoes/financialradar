Improve the UI layout of the Accounts page to group accounts by their type without changing any business logic, API calls, database schema, or existing state management.

Current situation:
All accounts are rendered in a single list, which looks messy when there are many accounts. Each account already has a `type` field (cash, bank, ewallet).

Goal:
Group the accounts visually by type so the UI looks like this:

Rekening

Bank
[ BNI ] Rp 5.125.421
[ BCA ] Rp 20.555

E-Wallet
[ Dana ] Rp 123.999
[ Gopay ] Rp 60.000
[ Shopee ] Rp 50.000

Tunai
[ Dompet ] Rp 23.500


Important rules:
- DO NOT change any API endpoints.
- DO NOT change database schema.
- DO NOT change account model or types.
- DO NOT change existing business logic for create/edit/delete accounts.
- Only refactor the UI rendering logic.
- Maintain existing account card component styling.
- Maintain existing edit and delete actions.
- Maintain mobile responsiveness.

Implementation approach:

1. After fetching the accounts list, group them by their `type` field in the frontend.

Example grouping logic:

const groupedAccounts = {
  bank: accounts.filter(a => a.type === "bank"),
  ewallet: accounts.filter(a => a.type === "ewallet"),
  cash: accounts.filter(a => a.type === "cash"),
};

2. Render sections conditionally only if that group has items.

Example structure:

{groupedAccounts.bank.length > 0 && (
  <section>
    <h3>Bank</h3>
    {groupedAccounts.bank.map(renderAccountCard)}
  </section>
)}

{groupedAccounts.ewallet.length > 0 && (
  <section>
    <h3>E-Wallet</h3>
    {groupedAccounts.ewallet.map(renderAccountCard)}
  </section>
)}

{groupedAccounts.cash.length > 0 && (
  <section>
    <h3>Tunai</h3>
    {groupedAccounts.cash.map(renderAccountCard)}
  </section>
)}

3. Keep the same card UI component for each account.

4. Add a small section header style for:
- Bank
- E-Wallet
- Tunai

Example style:

font-size: 14px
font-weight: 600
margin-top: 16px
margin-bottom: 8px
opacity: 0.8

5. Preserve current responsive layout:
- mobile: stacked list
- desktop: grid layout

6. Do not introduce new dependencies.

Expected result:
Accounts page becomes visually grouped by account type while keeping the same functionality.
