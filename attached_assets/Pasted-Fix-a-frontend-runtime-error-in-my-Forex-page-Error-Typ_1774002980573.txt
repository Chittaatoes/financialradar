Fix a frontend runtime error in my Forex page:

Error:
"TypeError: Cannot read properties of null (reading 'slice')"

This happens in production after deployment (Vercel), likely due to null/undefined values being used before data is ready.

---

## 🎯 GOAL

* Prevent all `.slice()` errors on null values
* Ensure safe rendering when data is not loaded yet
* Add fallback defaults for critical state (selectedPair, API response)

---

## 🔍 ROOT CAUSE

Some variables (like `selectedPair`, `pair`, or `data.pair`) are `null` or `undefined`, but are used like:

```ts
pair.slice(0, 3)
pair.split("/")
```

---

## 🔧 FIX IMPLEMENTATION

### 1. SAFE STRING ACCESS (GLOBAL FIX)

Replace ALL unsafe string usage:

❌ BEFORE:

```ts
pair.slice(0, 3)
pair.split("/")[0]
```

✅ AFTER:

```ts
pair?.slice(0, 3) || ""
pair?.split("/")?.[0] || ""
```

---

### 2. DEFAULT STATE (IMPORTANT)

Find in Forex page:

```ts
const [selectedPair, setSelectedPair] = useState(null);
```

Replace with:

```ts
const [selectedPair, setSelectedPair] = useState("EUR/USD");
```

---

### 3. RENDER GUARD

Wrap components that depend on data:

```tsx
if (!selectedPair) return null;
```

OR:

```tsx
{selectedPair && <ForexComponent />}
```

---

### 4. API RESPONSE SAFETY

Before using API data:

```ts
if (!data || !data.pair) return;
```

---

### 5. SAFE HELPER FUNCTION (RECOMMENDED)

Create utility:

```ts
function safeString(value?: string) {
  return value || "";
}
```

Use everywhere:

```ts
safeString(pair).slice(0, 3)
```

---

### 6. RADIX UI FIX (IMPORTANT)

Ensure all controlled components (Tabs, Select, Toggle) NEVER receive null:

❌

```tsx
<Tabs value={selectedPair}>
```

✅

```tsx
<Tabs value={selectedPair || "EUR/USD"}>
```
