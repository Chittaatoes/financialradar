Fix the custom budget cycle so that "Tanggal ke Tanggal" periods are fully persisted and used across the app.

Current problem:
The budget wizard correctly shows the custom cycle (example: "12 Mar – 11 Apr") in the confirmation step, but after saving the plan the budget page and dashboard revert to a monthly cycle ("01 Mar – 31 Mar").

Root cause:
The `upsertBudgetPlan` function in `storage.ts` does not update the fields:

* `cycleType`
* `cycleStartDay`

So when an existing plan is updated, these fields remain `"monthly"` even if the user selected `"date_to_date"` in the wizard.

Required fixes:

1. Fix persistence layer

In `upsertBudgetPlan` update branch, include the fields:

* `cycleType`
* `cycleStartDay`

Example structure:

update({
income,
strategy,
allocations,
cycleType,
cycleStartDay,
updatedAt: new Date()
})

This ensures the wizard properly saves the selected cycle.

2. Fix active period calculation

Wherever the app calculates the budget period (dashboard, budget summary, and daily budget calculations), change the logic:

If `cycleType === "monthly"`
→ period = first day of month → last day of month

If `cycleType === "date_to_date"`
→ period = cycleStartDay → next month cycleStartDay - 1

Example:

cycleStartDay = 12
Period should be:
12 Mar → 11 Apr

3. Ensure dashboard budget calculations follow the selected cycle

The following values must use the same computed cycle range:

* monthly income detection
* expense totals
* remaining monthly budget
* safe daily budget
* dashboard budget card numbers

Currently these still assume `1 → 31`.

4. Ensure UI period label uses the computed cycle

Budget card and budget page should display:

Periode Aktif
12 Mar – 11 Apr

instead of always showing the calendar month.

Important constraints:

* Do NOT change business logic for income detection or allocations.
* Only fix persistence of `cycleType` and `cycleStartDay`.
* Only update the period calculation so all financial calculations follow the selected cycle.
* Do NOT modify database schema.
* Do NOT change existing API routes.

Goal:

When a user selects "Tanggal ke Tanggal" and sets start date (example 12 March), the entire system (wizard summary, budget page, dashboard card, and calculations) must consistently use the cycle:

12 Mar → 11 Apr.
