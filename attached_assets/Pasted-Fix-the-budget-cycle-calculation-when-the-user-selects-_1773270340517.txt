Fix the budget cycle calculation when the user selects "tanggal ke tanggal" period.

Current problem:
When the user selects a custom cycle like 25 → 24, the system still calculates income, expense, and budget using the default monthly range (1 → 31).

This causes incorrect "budget aman harian" and incorrect totals.

Goal:
When the user selects a custom period (tanggal ke tanggal), all calculations must follow that cycle instead of the default calendar month.

Important constraints:
- Do NOT change business logic unrelated to budgeting.
- Do NOT change database schema.
- Do NOT modify API endpoints.
- Do NOT change transaction models.
- Only adjust how the date range is calculated for income, expense, and budget aggregation.

Required behavior:

If the user selects:

cycleStartDay = 25
cycleEndDay = 24

The active cycle must be calculated dynamically.

Example:

If today is:
2026-06-10

Then the active cycle should be:

2026-05-25 → 2026-06-24

If today is:
2026-06-28

Then the cycle should be:

2026-06-25 → 2026-07-24

Implementation logic:

Determine the cycle start and end dates based on today's date.

Example logic:

if (today.day >= cycleStartDay) {
  cycleStart = date(thisMonth, cycleStartDay)
  cycleEnd = date(nextMonth, cycleEndDay)
} else {
  cycleStart = date(previousMonth, cycleStartDay)
  cycleEnd = date(thisMonth, cycleEndDay)
}

Then all calculations must use:

WHERE date BETWEEN cycleStart AND cycleEnd

Apply this to:

- total income
- total expense
- remaining budget
- safe daily budget calculation
- remaining days in cycle

Example:

incomeCycle = sum(income transactions between cycleStart and cycleEnd)

expenseCycle = sum(expense transactions between cycleStart and cycleEnd)

remainingBudget = incomeCycle - expenseCycle

remainingDays = difference between cycleEnd and today

safeDailyBudget = remainingBudget / remainingDays


Important:
If the user selects the default monthly mode (1 → 31), keep the existing logic unchanged.

Expected result:

If the user chooses:

25 → 24

Then:

Income shown = income from 25 last month → 24 this month  
Expense shown = expense in the same range  
Safe daily budget = calculated using that cycle

This fix should only affect the date range logic used in calculations.
