Upgrade the Financial Radar web app to a complete offline-first architecture.

Goal:
The entire app must work even when the browser is offline.

Users must still be able to:

* create transactions (expense, income, transfer)
* scan receipts
* add accounts
* create goals
* record savings deposits
* record debt payments
* update budgets
* gain XP
* update streaks
* create custom categories
* use all UI features

When offline:
Actions must be saved locally.

When internet returns:
All changes must automatically sync to the backend API.

IMPORTANT:
Do not modify backend code or database schema.
All changes must be frontend only.

---

---

Create file:

frontend/src/lib/offline-sync.ts

Example stored item:

action: "create_transaction",
endpoint: "/api/transactions",
payload: {...},
createdAt: Date.now()
}

3. INTERCEPT ALL API REQUESTS

---

Modify:

frontend/src/lib/queryClient.ts

Intercept apiRequest() BEFORE fetch.

Logic:

if (!navigator.onLine)

store request in offline_queue

return a fake success response.

Do this for:

POST
PATCH
DELETE

Endpoints include:

/api/transactions
/api/accounts
/api/goals
/api/liabilities
/api/budget
/api/custom-categories
/api/streak
/api/profile
/api/daily-focus

Example behavior:

User creates a transaction while offline.

Instead of calling fetch:

saveOfflineAction({
endpoint: "/api/transactions",
payload: data
})

Return fake success.

5. AUTOMATIC SYNC WHEN ONLINE

---

Add listener:

syncOfflineQueue must:

1. read all offline_queue items
2. call API endpoints
3. delete queue items after success
